# E2GA for ISIC Conditional Diffusion Segmentation

This repository provides an implementation of **E2GA** for adversarial attacks on conditional diffusion-based medical image segmentation models.

The current released code is designed for the **ISIC skin lesion segmentation dataset** and is based on a MedSegDiff-style conditional diffusion segmentation model.

## Overview

Conditional diffusion models generate outputs through a stochastic multi-step reverse sampling process. Unlike deterministic discriminative models, the output of a conditional diffusion model depends not only on the conditional input but also on the initial noise and the stepwise random variables used during reverse denoising.

E2GA attacks the conditional input under an $L_\infty$ perturbation constraint. The attack estimates an input gradient through the reverse diffusion trajectory and updates the conditional image using a PGD-style procedure.

In this implementation, the attack is applied to ISIC images. The script saves:

- adversarial conditional images;
- segmentation outputs generated from the adversarial images.

## Supported Dataset

This release currently supports:

## Environment Setup

The code has been tested with Python 3.8.

Create a conda environment:

```bash
conda create -n e2ga python=3.8
conda activate e2ga
```

Install the dependencies:

```bash
pip install -r requirements.txt
pip install torchvision
```

> Note: Please make sure that `torch` and `torchvision` are compatible with your CUDA version.

## Data Preparation

Prepare the ISIC test images and place them under a directory such as:

```text
data/ISIC/Test/
```
The attack script reads ISIC images from the path specified by `--data_dir`.

## Running E2GA Attack

The full command is:

```bash
python scripts/attack_e2ga.py \
  --data_name ISIC \
  --data_dir ./data/ISIC/Test/ISBI2016_ISIC_Part1_Test_Data \
  --out_dir outputs/e2ga_isic \
  --model_path emasavedmodel_0.9999_580000.pt \
  --image_size 256 \
  --num_channels 128 \
  --class_cond False \
  --num_res_blocks 2 \
  --num_heads 1 \
  --learn_sigma False \
  --use_scale_shift_norm False \
  --attention_resolutions 16 \
  --diffusion_steps 1000 \
  --noise_schedule linear \
  --rescale_learned_sigmas False \
  --rescale_timesteps False \
  --num_ensemble 1 \
  --batch_size 1 \
  --epsilon 0.031372549 \
  --step_size 0.0039215686 \
  --attack_steps 20
```

If these model parameters have already been set as defaults in `scripts/attack_e2ga.py`, the command can be simplified as:

```bash
python scripts/attack_e2ga.py \
  --data_dir ./data/ISIC/Test/ISBI2016_ISIC_Part1_Test_Data \
  --model_path emasavedmodel_0.9999_580000.pt \
  --out_dir outputs/e2ga_isic
```

Here:

- `epsilon=0.031372549` corresponds to an $8/255$ perturbation budget;
- `step_size=0.0039215686` corresponds to a $1/255$ PGD step size;
- `attack_steps=20` means 20 adversarial update steps.

## Output Files

After running the attack, the results will be saved under:

```text
outputs/e2ga_isic/
├── adv_images/
│   └── *_adversarial.png
└── adv_output/
    └── *_output_ens.png
```

The folder `adv_images/` contains adversarially perturbed ISIC images.

The folder `adv_output/` contains the corresponding segmentation outputs generated by the diffusion model.
